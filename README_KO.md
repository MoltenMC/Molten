# Molten

Molten은 공유 ECS 게임 코어와 독립적인 Java Edition 및 Bedrock Edition 프로토콜 스택을 중심으로 설계된 엔터프라이즈급 하이브리드 마인크래프트 서버 엔진입니다.

## 목표

- Java Edition 프로토콜 776.
- Bedrock Edition 프로토콜 1001.
- Java 언어/툴체인 레벨 25를 대상으로 하는 Kotlin 구현.
- 플랫폼 독립적인 게임플레이 로직을 `molten-common`에 배치하는 멀티 모듈 Gradle 아키텍처.

## 모듈

- `molten-common`: ECS, 레지스트리, 스케줄러, 메모리, 수학, 직렬화, 공유 월드 모델 기반.
- `molten-api`: 플러그인 대상 공개 API.
- `molten-server`: 서버 부트스트랩, 라이프사이클, 런타임 모듈 로딩, 설정, 플러그인 로딩 진입점.
- `molten-java`: Java Edition 프로토콜 파이프라인 및 어댑터.
- `molten-bedrock`: Bedrock Edition 프로토콜 파이프라인 및 어댑터.
- `molten-translator`: 레지스트리 및 프로토콜 변환 레이어.

## 아키텍처 원칙

- 핵심 게임플레이는 프로토콜에 독립적으로 유지됩니다.
- 프로토콜 구현은 API/common 계약 방향으로 내부 의존합니다.
- 플랫폼 스토리지 및 트랜스포트 구현은 SPI 스타일 인터페이스 뒤에서 교체 가능합니다.
- ECS 구조적 변경은 커맨드 버퍼를 통해 지연 처리됩니다.

## 런타임 모드

- `JAVA_BASED`: Java 시맨틱 기반, Java 및 Bedrock 클라이언트 활성화, Anvil 기본 스토리지.
- `JAVA_ONLY`: Java 클라이언트만 지원, Anvil 기본 스토리지.
- `BEDROCK_BASED`: Bedrock 시맨틱 기반, Java 및 Bedrock 클라이언트 활성화, LevelDB 기본 스토리지.
- `BEDROCK_ONLY`: Bedrock 클라이언트만 지원, LevelDB 기본 스토리지.

## ECS 방향

- 엔티티 ID는 64비트 레이아웃을 사용합니다: 32비트 인덱스, 24비트 세대, 8비트 종류.
- 안정적인 컴포넌트는 아키타입 스토리지용으로 모델링됩니다.
- 휘발성/태그형 컴포넌트는 스파스 셋 스토리지용으로 모델링됩니다.
- 구조적 변경은 커맨드 버퍼와 결정론적 플러시 포인트를 통해 흐릅니다.
- 시스템은 디스크립터를 통해 읽기/쓰기 접근을 선언하여 스케줄러가 안전한 병렬 실행을 추론할 수 있도록 합니다.

## 네트워크 방향

- Java Edition과 Bedrock Edition은 독립적인 패킷 레지스트리, 코덱 계약, 세션 상태, 연결 흐름을 가집니다.
- 네트워크 핸들러는 검증된 패킷을 플랫폼 중립적인 서버 인텐트로 변환하며, 게임플레이 변경은 시뮬레이션 리전 내부에서만 이루어집니다.
- 고빈도 패킷 경로는 지연 패킷 뷰를 사용할 수 있고, 저빈도 패킷은 강타입 패킷 객체로 디코딩할 수 있습니다.
- 레지스트리 및 변환 로직은 Java와 Bedrock 프로토콜 데이터를 플랫폼 중립적인 Molten 모델로 매핑한 후 공통 게임플레이 코드에 전달합니다.
- 네트워크 트래픽은 신뢰할 수 없는 것으로 취급되며, 명시적 패킷 검증, 속도 제한, 크기 제한, 비정상 패킷 정책을 통해 검사됩니다.

## API 및 플러그인

- `molten-api`는 플레이어, 월드, 엔티티, 이벤트, 커맨드, 권한, 스케줄링, 컴포넌트, 시스템을 위한 안정적인 고수준 플러그인 API를 제공합니다.
- 고급 확장 포인트는 `molten-common` SPI 계약으로, 프로토콜 어댑터, 패킷 코덱, 레지스트리, 스토리지, 변환기, 권한, 스케줄러 관찰, 메트릭 내보내기를 포함합니다.
- 플러그인은 `molten-plugin.yml` 메타데이터와 공유 API 계약을 가진 격리 클래스로더 모델을 사용합니다.
- 상태 변경 이벤트는 동기적이며 리전 바운드입니다. 관찰 이벤트는 비동기적으로 실행될 수 있습니다.
- 커맨드는 Molten 커맨드 트리에 한 번 등록되며 Java Brigadier 또는 Bedrock 커맨드 데이터로 내보낼 수 있습니다.

## 엔지니어링 게이트

- GitHub Actions에서 풀 리퀘스트 및 `main` 브랜치에 대해 `./gradlew --no-configuration-cache clean build`를 실행합니다.
- 모듈 전반에 걸쳐 Kotlin 테스트가 활성화되어 있습니다.
- 초기 단위 테스트는 엔티티 ID 패킹, 스파스 셋 동작, 권한 해석을 다룹니다.
- 성능 및 로드맵 열거형은 구현 의존성을 노출하지 않으면서 계획된 벤치마크, 메트릭, 확장 영역을 포착합니다.
